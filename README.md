# 자동차 경주 게임

## 기능 요구사항

### 1,2단계. 자동차 움직임 및 우승자 결정(Car, Cars)
- [x] 자동차는 이름을 가진다. (5자 이하 제한)
- [x] 무작위 값이 4 이상일 경우 전진, 3 이하일 경우 정지한다.
- [x] 게임 완료 후 가장 멀리 간 자동차(들)를 우승자로 판별한다.

### 3단계. 게임 실행 (Controller & UI)
- [x] 사용자에게 자동차 이름(쉼표 구분)과 시도 횟수를 입력받는다.
- [x] 매 라운드마다 자동차들의 현재 위치(`-`)를 출력한다.
- [x] 최종 우승자를 출력한다. (공동 우승 시 `, `로 구분)

### 4단계. 리팩터링 및 테스트 
- [x] **모든 도메인 로직에 대한 단위 테스트를 작성한다.**
- [x] **UI 로직(System.in, out)을 제외한 비즈니스 로직만 테스트한다.**
- [x] **랜덤한 요소(NumberGenerator)를 제어하여 이동/정지를 확정적으로 테스트한다.**

## 프로그래밍 요구사항 (제약 사항)
- **메인 메서드를 추가하여 실행 가능한 애플리케이션으로 만든다.**
- 값을 입력받기 위해 `Scanner`를, 랜덤 값 생성을 위해 `java.util.Random`을 사용한다.
- Java Style Guide를 준수한다.
- **indent(들여쓰기) depth는 1까지만 허용한다.** (메서드 분리를 적극 활용)
- **3항 연산자를 쓰지 않는다.**
- **else 예약어(switch 포함)를 쓰지 않는다.** - **메서드 길이는 15라인을 넘어가지 않도록 구현한다.** ```
